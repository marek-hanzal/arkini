# Terse output effect copy

Active production effects are now displayed as short output-scoped outcome lines instead of source/effect descriptions.

Rules:
- Do not describe which nearby/source effect caused the bonus in the output row.
- Show only what the bonus does to that output.
- Use terse labels such as `Speed: 27% faster` and `Drop: +2× guaranteed, 20% chance for +1×`.
- Output rows already provide the affected item context, so do not repeat the item name in bonus lines.
- Keep source/effect labels for blocked/requirement/debug views, not for applied output bonus copy.
