# Documentation archive

This directory contains completed reviews, superseded designs, historical task queues, removed deployment files, and other repository archaeology.

Nothing here is an active contract.

When an archived document conflicts with root documentation, current project memory, tests, or source, the archived document loses. Its purpose is to preserve decision history, not to instruct new implementation work.

Search this directory only when investigating why a current rule exists or when reconstructing historical context for a regression.
